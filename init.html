Classes para regras de validação

As classes listadas abaixo são as classes padrões usadas pelo plugin para aplicar as regras de validação correspondente. As classes seletoras podem ser customizadas. Veja a página de configurações para mais detalhes.

Classe Descrição Exemplo
required Não permite campos vazios. Pelo menos um campo checkbox e radio com mesmo nome deve ser selecionado.
email Permite apenas emails válidos.
integer Permite apenas números.
digit Permite apenas letras e números.
url Permite apenas urls válidas.
minlength Permite apartir de uma quantidade mínima configurada. A configuração pode ser feita usando o atributo data-minlength.
maxlength Permite até uma quantidade máxima configurada. A configuração pode ser feita usando o atributo data-maxlength.
rangelength Permite até uma quantidade máxima configurada e apartir de uma quantidade mínima configurada. A configuração pode ser feita usando os atributos data-minlength e data-maxlength.
min Permite um valor inteiro maior ou igual que o valor configurado. A configuração pode ser feita usando o atributo data-min.
max Permite um valor inteiro menor ou igual que o valor configurado. A configuração pode ser feita usando o atributo data-max.
range Permite um valor inteiro menor ou igual que o valor configurado e um valor inteiro maior ou igual que o valor configurado. A configuração pode ser feita usando os atributos data-min e data-max.
equalTo Compara o valor do elemento com o valor do elemento alvo. Devem ser iguais. O elemento alvo é configurado no atributo data-target, onde você informa a classe do elemento alvo.
date Permite apenas datas válidas. Formato brasileiro 99/99/9999
cpf Permite apenas cpf válido.
cnpj Permite apenas cnpj válido.
cep Permite apenas ceps válidos. Formato brasileiro: 99999-999, 99.999-999 ou 99999999
phone Permite apenas números de telefone válidos. Formato brasileiro: 21 9 9999 9999
creditcard Permite apenas números de cartão válidos.
extension Permite apenas as extensões de arquivos configurados. A configuração pode ser feita usando o atributo data-extensions.
